## Purpose

This document tracks the scheduled review dates for all ISMS policies and procedures, ensuring compliance with NIST CSF 2.0 subcategory **GV.PO-02** (policy is reviewed, updated, communicated, and enforced).

---
## Review Schedule

| Document | Review Frequency | Q1 (Jan–Mar) | Q2 (Apr–Jun) | Q3 (Jul–Sep) | Q4 (Oct–Dec) |
|----------|-----------------|:---:|:---:|:---:|:---:|
| Acceptable Use Policy | Quarterly | ☐ | ☐ | ☐ | ☐ |
| Incident Response Plan | Quarterly + post-incident | ☐ | ☐ | ☐ | ☐ |
| NIST CSF Crosswalk | Quarterly | ☐ | ☐ | ☐ | ☐ |
| RACI Matrix | Semi-annually | ☐ | - | ☐ | - |
| Data Flow Diagram | Semi-annually | ☐ | - | ☐ | - |
| Risk Appetite Statement | Annually | ☐ | - | - | - |
| This Review Cadence | Annually | ☐ | - | - | - |

**How to use:** Check the box (☐ → ☑) when the review is completed. Update the document's YAML `version` and `date` fields after each review.

---
## Trigger-Based Reviews
In addition to scheduled reviews, the following events trigger an immediate review:

| Trigger | Documents Affected |
|---------|-------------------|
| Security incident occurs | IRP, Crosswalk, Risk Assessment |
| Regulatory change | All policies, Crosswalk |
| Significant IT environment change | DFD, Crosswalk, AUP |
| Tabletop or live exercise completed | IRP, Ransomware Playbook |
| Organizational restructuring | RACI Matrix, AUP |
